## Core Principles: Trust & Control

### 🔒 Principle 1: Data Sovereignty (Your Data Stays HOME)

Your data NEVER leaves your machine unless you explicitly ask a question.

```
┌─────────────────────────────────────────────────────────────────┐
│                    DATA FLOW ARCHITECTURE                       │
│                                                                 │
│   YOUR PC                           │  EXTERNAL                 │
│   ─────────────────────────────────│────────────────────────── │
│                                     │                           │
│   📁 Files          ────┐           │                           │
│   📧 Emails         ────┤           │                           │
│   📅 Calendar       ────┼── LOCAL ──│── NEVER ──X              │
│   📇 Contacts       ────┤   INDEX   │                           │
│   📄 Documents      ────┘           │                           │
│                                     │                           │
│   🧠 Semantic Memory (local)        │                           │
│   📊 Pattern Learning (local)       │                           │
│   🔍 Search Index (local)           │                           │
│                                     │                           │
│   ─────────────────────────────────│────────────────────────── │
│                                     │                           │
│   You: "What's in the loan doc?"    │                           │
│         ↓                           │                           │
│   [Only YOUR question + minimal     │   🤖 LLM                  │
│    context goes out]─────────────────────────>  (processes)    │
│         ↓                           │         ↓                 │
│   [Answer comes back]<───────────────────────────              │
│                                     │                           │
└─────────────────────────────────────────────────────────────────┘
```

**What stays local:**
- All your files, emails, calendar events, contacts
- Semantic memory and learned patterns
- Search indexes and entity relationships
- Draft documents and pending actions

**What goes to the LLM (only when you ask):**
- Your question/request
- Minimal relevant context (summaries, not raw data)
- NEVER: passwords, full documents, personal files
